git学习

 尽管已经接触并使用github许久了，但对git的命令还是不太熟，于此记录下git指令.

本地操作

  1. 配置命令如下：
git config --global[/system/local/worktree] -l  # 查看所有配置
git config --global[/system/local/worktree] --add[/unset] {key} # 添加[/删除]配置键名
git config --global[/system/local/worktree] --get {key} # 获取单个配置

 更详细的通过 git config -h 获取;

  1. 对本地仓库操作：
git init [dir] # 初始化当前[dir]目录
git status [dir] # 查看当前[dir]目录仓库暂存区状态
git add {file/dir}  # 添加文件或目录到暂存区
git rm --cached {file}  # 删除暂存区的文件
git commit -m {msg} {file}  # 提交file文件并附上消息msg
git reflog  # 查看引用日志
git log # 查看详细日志
git reset {version} # 回到对应版本

注意
 首次提交，此时还没有master分支，通过git commit -m {msg}后会提交并自动创建分支;

  1. 分支操作命令：
git branch -v # 查看分支
git branch {name} # 新建名为name分支
git checkout {name} # 切换到name分支
git merge {name}  # 将name分支合并到当前分支

 分支出现冲突，即多个分支都对同一文件修改并都已提交，需要手动修改文件并添加到暂存区与提交.

团队协作

 接下来为git团队协作，请参考：https://github.com/geeeeeeeeek/git-recipes

团队内协作

  1. 远程仓库操作，创建和推送远程库
git remote -v # 查看所有远程库地址别名
git remote add {name} {addr}  # 创建别名为name远程仓库
git push {remote} {branch}  # 将本地branch分支推送到name远程库

 补充，github现在更新策略采用密钥登录，需要为仓库添加访问公钥，下面为获取公钥的命令：

ssh-keygen -t rsa # 如果没有公钥，通过该命令生成
cat /root/.ssh/id_rsa.pub # 查看公钥

 将获取的公钥添加到仓库 -> Settings -> Security -> Deploy keys -> Add deploy key;
 勾选允许写权限;

  1. 获取、合并和克隆远程库
git pull {remote} {branch}  # 将remote远程库同步到本地branch分支，等于fetch和merge，可能会影响到本地库，不安全
git fetch {remote} {branch} # 获取remote远程指定分支的所有提交，但并不merge，较安全
git merge {remote}/{branch} # 合并remote远程库的branch分支到当前分支
git clone {addr} [dir]  # 克隆addr仓库到当前[dir]目录
  1. 撤销提交
git reset {file}  # 从缓存区移除特定文件，但不改变工作区
git reset {commit}  # 重设缓冲区，匹配commit对应的提交，但工作目录不变
git revert {commit} # 撤销commit对应的提交，不会改变项目历史，更安全

跨团队协作

  1. fork某个仓库
  2. 对fork的仓库修改后并提交
  3. 在Pull request中Create Pull request
  4. 主仓库对代码审核，决定是否merge

其它

指令区别

  1. pull和fetch

  2. reset和revert

 参考:https://github.com/geeeeeeeeek/git-recipes

多用户使用指定ssh私钥

 在远程仓库中添加部署公钥后，如果有多个帐号，需要指定特定的私钥，修改~/.ssh/config文件:

$ cat ~/.ssh/config
# remote地址比如: git@用户名1.gitee.com:用户名1/test.git
# 而不是git@gitee.com:用户名1/test.git，否则多个gitee帐号可能无法push
Host 用户名1.gitee.com
  User 用户名1 # 用户名
  Hostname gitee.com
  PreferredAuthentications publickey
  IdentityFile /to/path/id_rsa.用户名1.gitee # 私钥路径，私钥文件名自定义
  IdentitiesOnly yes
Host 用户名2.gitee.com
  User 用户名2
  Hostname gitee.com
  PreferredAuthentications publickey
  IdentityFile /to/path/id_rsa.用户名2.gitee
  IdentitiesOnly yes
Host 用户名3.github.com
  User 用户名3
  Hostname github.com
  PreferredAuthentications publickey
  IdentityFile /to/path/id_rsa.用户名3.github
  IdentitiesOnly yes
#Host *  # 默认ssh私钥。最好注释否则会影响到Jetbrains软件的ssh登录
#  User 用户名4
#  Hostname *
#  PreferredAuthentications publickey
#  IdentityFile /to/path/id_rsa
#  IdentitiesOnly yes

 以上仅是通过配置文件的示例，或可以通过命令，可以参考：https://hanpanpan200.github.io/2019/10/14/setup-multiple-git-accounts-by-ssh-config/

注意:

 上方注释的部分即 ‘ Host * ‘ 的部分，算是一个坑，使用IDEA的ssh测试连接时，一直报错，此前连接正常，无修改服务器配置，报错如下:

Connection to xxx@xxx:xx
Cannot connect to remote host: host not found

 经过测试，使用winterm和命令行都能正常登录，但IDEA2022.2.4和旧版都不能ssh登录;

 在这篇文章中也有提到，可以确定时是软件的bug.

 解决办法是注释掉Host *等内容.

首次提交或已存在仓库

  1. 针对新建仓库第一次提交
mkdir test
cd test
git init 
touch README.md
git add README.md
git commit -m "first commit"
git remote add origin git@gitee.com:test/test.git
git push -u origin "master"
  1. 已有本地仓库进行首次提交;
cd existing_git_repo
git remote add origin git@gitee.com:test/test.git
git push -u origin "master"
