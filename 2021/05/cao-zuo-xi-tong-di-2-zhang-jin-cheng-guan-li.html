进程调度

调度的基本原则

 为了衡量不同调度算法的特性，需要有一些评价标准：

  1. CPU利用率

  2. 系统吞吐量
     单位时间内CPU完成作业的数量；

  3. 响应时间
     从用户出发，要尽可能减小响应时间；

  4. 周转时间

  • 周转时间
     周转时间指作业从提交至完成的时间间隔，包括等待时间和执行时间。
     周转时间 Ti 公式表示为：

    作业i的周转时间 Ti = 作业 i 的完成时间 - 作业 i 的提交时间

  • 平均周转时间
     平均周转时间指多个作业(例如n个作业)周转时间的平均值，用公式可以表示为：

    T = ( T1 + T2 + ... + Tn) / n

  • 带权周转时间
     带权周转时间指作业周转时间于运行时间的比，公式表示为：

    Wi = 作业 i 的周转时间/作业 i 的运行时间

  • 平均带权周转时间
     与平均周转时间类似，是多个作业的带权周转时间平均值。略

常见调度算法

  1. 先来先服务调度算法(FCFS)
     按照进程先后次序分配处理器；略

  2. 短作业优先调度算法(SJF)
     执行时间短的优先被分配；长作业可能会出现“饥饿”现象；

  3. 优先级调度算法
     可以分为静态优先级和动态优先级调度；

  • 静态优先级调度
     静态优先级调度是在创建进程时确定的，确定之后在整个进程运行期间不再改变；
     确定优先级的依据有：按进程类别，按作业的资源要求，按用户类型和要求等；

  • 动态优先级调度
     动态优先级调度指在创建进程时，根据进程的特点及相关情况确定优先级，在进程运行过程中根据情况的变化调整优先级；
     确定动态优先级依据有：根据进程占有CPU时间的长短，根据就绪进程等待CPU时间的长短等；
     还可以按照调度方式的不同划分为非抢占式和抢占式优先调度。

  1. 时间片轮转调度算法
     时间片过大，会退化为FCFS算法；过小，进程会频繁切换，真正用于运行进程的时间将会减小。

  2. 高响应比优先调度算法

  3. 多级队列调度算法

  4. 多级反馈队列调度算法

进程同步互斥

 这里略去软件方法实现互斥

  1. 信号量
     定义
    struct semaphore{         //信号量有二元组组成，整型变量和等待队列
  int count;
  queueType queue;
}
wait(semaphore s){        //P操作
  s.count--;
  if(s.count<0>){
    阻塞该进程;
    将该进程插入等待队列s.queue;
  }
}
signal(semaphore s){      //V操作
  s.count++;
  if(s.count<=0){
    从等待队列s.queue中取出第一个进程P;
    将P插入就绪队列;
  }
}

 P操作相当于申请资源，V操作相当于释放资源，P操作和V操作一定是成对出现，但不一定在一个进程中，可以分布在不同进程中。

  1. 经典同步问题
  • 生产者-消费者问题

 生产者产生进程，消费者释放进程

semaphore full=0;         //满缓冲区数目
semaphore empty=n;        //空缓冲区数目
semaphore mutex=1;        //对有界缓冲区进行操作的互斥信号量
Producer(){
  while(true){
    Produce an item put in nextp; //nextp为临时缓冲区
    P(empty);             //申请一个空缓冲区
    P(mutex);             //申请使用缓冲池
    将产品放入缓冲池
    V(mutex);             //释放缓冲池信号量
    V(full);              //增加一个满缓冲区
  }
}

Consumer(){
  while(true){
    P(full);              //申请一个满缓冲区
    P(mutex);             //申请使用缓冲池
    取出产品;
    V(mutex);             //释放缓冲池信号量
    V(empty);             //增加一个空缓冲区
    Consume the item in nextc;  //消费
  }
}

注意
 P(full) / P(empty)与P(mutex)顺序不能颠倒，必须先对资源信号量进行P操作，再对互斥信号量进行P操作，否则会导致死锁；而对V操作顺序没有要求；
 几乎所有P、V操作在有多个信号量同时存在的情况下，P操作往往不能颠倒顺序，必须先对资源信号量进行P操作，再对互斥信号量进行P操作，这样可以在占有信号量访问权时保证有资源可以使用，否则会产生占用使用权而无资源可用的“死等”现象。

  1. 读者-写者问题
     在读者-写者问题中，有一个共享的数据区，读进程与写进程对数据区访存需要满足一定条件：

1.）任意多个读者可以同时读这个数据区；
2.）一次只能有一个写者往数据区写(写者必须互斥)
3.）如果一个写者正在操作，禁止其他进程对数据区操作。

  • 读者优先算法

    semaphore rmutex=1;       //保证对readcount互斥访问
semaphore mutex=1;        //保证对数据区互斥访问
int readcount=0;          //读者数量
reader(){
  while(true){
    P(rmutex);            //申请readcount使用权
    if(readcount==0)      //为第一个读者，阻止写者进入
      P(mutex);
    readcount++;          //读者数量加一
    V(rmutex);            //释放readcount使用权
    进行读操作;
    P(rmutex);            //申请readcount使用权
    readcount--;          //读者数量减一
    if(readcount==0)      //没有读者，允许写者进入
      V(mutex);
    V(rmutex);            //释放readcount使用权
  }
}

writer(){
  while(true){
    P(mutex);             //申请对数据区访问
    进行写操作;
    V(mutex);             //释放对数据区访问权
  }
}

  • 公平情况算法(按照到达顺序进行操作)

semaphore mutex=1;        //互斥访问数据区
semaphore rmutex=1;       //互斥访问readcount
semaphore wmutex=1;       //存在写者时禁止新读者进入，为1时表示没有写者
int readcount=0;          //读者数量
reader(){
  while(true){
    P(wmutex);            //检测写者是否存在
    P(rmutex);            //申请访问readcount
    if(readcount==0)      //只有一个读者，阻止写者进入
      P(mutex)
    readcount++;          //读者数量加一
    V(rmutex);            //释放对readcount的访问权
    V(wmutex);            //恢复wmutex
    进行读操作;
    P(rmutex);            //申请使用readcount使用权
    readcount--;          //读者数量减一
    if(readcount==0)      //没有读者，允许写者进入
      V(mutex);
    V(rmutex);            //释放readcount使用权
  }
}

writer(){
  while(true){
    P(wmutex);            //检测有无写者，没有写者时进入
    P(mutex);             //申请访问数据区
    进行写操作;
    V(mutex);             //释放对数据区访问
    V(wmutex);            //恢复wmutex
  }
}
  • 写者优先算法
semaphore mutex=1;        //数据区互斥访问信号量
semaphore rmutex=1;       //读者互斥访问信号量
semaphore wmutex=1;       //写者互斥访问信号量
semaphore readable=1;     //读者可读
int readcount=0, writecount=0;  //读者数量，写者数量
reader(){
  P(readable);            //检测是否可读
  P(rmutex);              //申请对readcount访问权
  if(readcount==0)        //第一个读者，占用数据区访问权
    P(mutex);
  readcount++;            //读者数加一
  V(rmutex);              //释放rmutex，允许其他读者访问readcount
  V(readable);            //释放readable，允许其他读者，写者占用
  执行读操作;
  P(rmutex);              //占用rmutex，准备修改readcount
  readcount--;            //读者数量减一
  if(readcount==0)        //没有读者，释放对数据区访问权
    V(mutex);
  V(rmutex);              //释放rmutex
}

writer(){
  P(wmutex);              //占用wmutex
  if(writecount==0)       //第一个写者，阻止读者进入
    P(readable);
  writecount++;           //写者数量加一
  V(wmutex);              //释放wmutex
  P(mutex);               //占用数据区
  执行写操作;
  V(mutex);               //释放数据区
  P(wmutex);              //释放wmutex
  writecount--;           //写者数量减一
  if(writecount==0)       //没有读者，允许读者进入
    V(readable);          //释放wmutex
  V(wmutex);
}

  1. 理发师问题

 理发店只有一个理发师，一张理发时坐的椅子，若干张普通椅子顾客供等候时坐。没有顾客时，理发师就坐在理发的椅子上睡觉。顾客一到，他不是叫醒理发师，就是离开。如果理发师没有睡觉，而在为别人理发，他就会坐下来等候。如果所有的椅子都坐满了人，最后来的顾客就会离开。

  • 第一种思路
     分别考虑理发椅和凳子：
int waiting=0;                //等待的人数
semaphore mutex=1;            //互斥信号量
semaphore bchair=1;           //里发互斥信号量，用于占有理发椅的使用权
semaphore wchair=n;           //椅子互斥信号量，用于互斥占用椅子
semaphore ready=finish=0;     //就绪信号量与完成信号量
barber(){
  while(true){
    P(ready);                 //从就绪队列中申请一个客户理发
    理发;
    V(finish);                //完成，可以为一个用户理发
  }
}

customer(){
  P(mutex);                   //申请使用waiting变量
  if(waiting<=n>){            //如果还有空位
    waiting++;                //等待人数加一
    V(mutex);                 //释放对waiting访问权
  }else{
    V(mutex);                 //椅子已经坐满，释放waiting访问权
    离开;
  }
  P(wchair);                  //申请椅子
  P(bchair);                  //申请理发椅，等待理发椅空闲后占用
  V(wchair);                  //释放椅子使用权
  V(ready);                   //使用理发椅，放入就绪队列
  P(finish);                  //等待理发完成
  V(bchair);                  //释放理发椅占有权
  P(mutex);                   //申请访问waiting
  waiting--;                  //等待人数建议
  V(mutex);                   //释放对waiting访问
}
  • 第二种思路
     将理发椅与凳子看作一个变量
int chairs=n+1;               //椅子和理发椅共 n+1 个
semaphore ready=0;            //就绪信号量
semaphore finish=1;           //完成信号量
semaphore mutex=1;            //互斥信号量
barber(){
  while(true){
    P(ready);                 //从就绪队列中申请给一个客户理发
    理发;
    P(mutex);                 //申请访问chairs变量
    chairs++;                 //理完发，空的椅子加一
    V(mutex);                 //释放对chairs的访问
    V(finish);                //完成理发，可以为下一个用户理发
  }
}

customer(){
  P(mutex);                   //申请访问chairs变量
  if(chairs>0){               //如果有空座位
    chairs--;                 //空座位减一
    V(mutex);                 //释放访问chairs
    V(ready);                 //就绪
    P(finish);                //等待上一个客户理完发
  }else{
    V(mutex);                 //没有座位，释放访问chairs
  }
}

死锁

死锁的原因和必要条件

  • 原因：系统资源不足和进程推进顺序不当；
  • 必要条件：

1.互斥条件
 进程对所分配的资源进行排他性控制，即在一段时间内某种资源仅为一个进程所占有；

2.不剥夺条件
 进程所获得的资源在未使用完毕之前，不能被其他进程强行夺走；

3.请求与保持条件
 进程每次申请它所需的一部分的资源。在等待分配新资源的同时，进程继续占有已经分配到的资源。请求与保持条件也称为部分分配条件。

4.环路等待条件
 存在一种进程资源的循环等待链，而链中的每个进程已经获得的资源同时被链中的下一个进程所请求。

处理死锁的基本方法

  • 鸵鸟算法
     即对死锁视而不见；

  • 预防死锁
     通过设置某些限制条件，来破坏产生死锁的4个必要条件中的一个或几个；

  • 避免死锁
     在资源的动态分配过程中，用某种方法防止系统进入不安全状态，从而避免死锁的产生。

  • 检测及解除死锁
     通过系统的检测机构及时检测出死锁的发生，然后采取某种措施解除死锁。

银行家算法

 假定系统中有 n 个进程(P1, P2, P3, … , Pn)和m类资源(R1, R2, R3, … , Rm)

  • 需要用的数据结构
  1. 可利用资源向量Available
    Available[i] = 第i类资源的现有空闲数量
     它的数值会随着资源的分配和回收动态改变；

  1. 最大需求矩阵Max

    Max[i][j] = 第i个进程对第j类资源的最大需求数

  2. 分配矩阵Allocation

    Allocation[i][j] = 第i个进程当前拥有的第j类资源的数量

  3. 需求矩阵Need

    Need[i][j] = 第i个进程还需要第j类资源的数量

 从上面的矩阵中可以得出一个关系：

Need[i][j] = Max[i][j] - Allocation[i][j]
  • 算法描述

 定义Requesti向量：Requesti表示第i个进程向系统提出一次申请，申请的各资源的数量就是各个分类。
 当进程向系统发出资源请求后，系统进行如下操作：

1.）若Requesti ≤ Needi，则跳转至2.否则报错，因为进程Pi申请的资源数不应该是超过它的需求数；
2.）若Requesti ≤ Availablei，则跳转至3.否则Pi进程需要等待，因为可用资源不够；
3.）对Pi进程所请求的资源进行预分配，修改以下向量：
 Availablei = Availablei-Requesti
 Allocationi = Allocationi+Requesti
 Needi = Needi-Requesti
4.）对于修改后的向量调用安全性算法
 若安全性算法返回系统处于安全状态，则按Requesti表示的资源数量给Pi进程分配资源；若安全性算法返回系统处于不安全状态，则分配给Pi进程任何资源，让Pi等待，并恢复 3.）中所改变的向量。

 安全算法描述如下：

1.）建立长度为m的向量Work和长度为n的向量Finish，并对它们进行如下初始化：Work = Available；Finish[i] = false；（i = 1, 2, … , n)。
2.）查找满足如下两个条件的 i ：
 Finish[i] = false； Need[i] ≤ Work；
 若没有这样的 i ，则跳到 4.）执行；
3.）进行如下操作：
 Work = Work + Allocation[i]
 Finish[i] = true
返回到 2.）
4.）如果对于所有 i ，都有Finish[i] = true，那么系统处于安全状态，否则为不安全状态。
