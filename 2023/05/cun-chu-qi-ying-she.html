存储器映射

 STM32芯片为STM32F103ZET6，下面介绍存储器映射与封装;

 存储器的地址需要由芯片厂商或用户分配，给存储器分配地址的过程就叫做存储器映射，如果再分配一个地址就叫做重映射，STM32F103ZET6芯片的存储器映射如下图所示:

 存储器区域划分: ARM将该芯片的4G存储空间平均划分成8块，如下所示:

序号用途地址范围
Block0SRAM(Flash)0x0000 0000 ~ 0x1FFF FFFF(512MB)
Block1SRAM0x2000 0000 ~ 0x3FFF FFFF(512MB)
Block2片上外设0x4000 0000 ~ 0x5FFF FFFF(512MB)
Block3FSMC的bank1~bank20x6000 0000 ~ 0x7FFF FFFF(512MB)
Block4FSMC的bank3~bank40x8000 0000 ~ 0x9FFF FFFF(512MB)
Block5FSMC register0xA000 0000 ~ 0xBFFF FFFF(512MB)
Block6Not used0xC000 0000 ~ 0xDFFF FFFF(512MB)
Block7Cortex-M3内部外设0xE000 0000 ~ 0xFFFF FFFF(512MB)

 在8个Block中，Block0、Block1和Block2是应该关心的;

  • Block0内部区域功能划分
地址作用
0x0000 0000 ~ 0x0007 FFFF取决于额BOOT引脚，为Flash、系统存储器、SRAM的别名
0x0008 0000 ~ 0x07FF FFFF预留
0x0800 0000 ~ 0x0807 FFFF片内Flash，存放编写的程序
0x0808 0000 ~ 0x1FFF EFFF预留
0x1FFF F000 ~ 0x1FFF F7FF系统存储器，存放出厂时程序
0x1FFF F800 ~ 0x1FFF F80F选项字节(读写保护、BOR级别、软硬件看门狗、复位)
0x1FFF F810 ~ 0x1FFF FFFF预留
  • Block1内部区域功能划分
地址作用
0x2000 0000 ~ 0x2000 FFFFSRAM，容量64KB
0x2001 0000 ~ 0x3FFF FFFF预留
  • Block2内部区域功能划分
地址作用
0x4000 0000 ~ 0x4000 77FFAPB1总线外设
0x2001 7800 ~ 0x4000 FFFF预留
0x4001 0000 ~ 0x4001 3FFFAPB2总线外设
0x4001 4000 ~ 0x4001 7FFF预留
0x4001 8000 ~ 0x4002 33FFAHB总线外设
0x4002 4400 ~ 0x5FFF FFFF预留

寄存器和寄存器映射

 由于Cortex-M3内核是32位，每4个字节为一个单元，每一个单元对应不同的功能，通过控制这些单元从而控制外设;由于外设非常多、复杂，意味着需要操作较多的存储单元地址，因此将每个单元的功能作为名字，给其取别名，别名就是寄存器;

 将寄存器名映射到单元的过程称为寄存器映射;

外设地址映射

 APB1挂载低速外设，APB2和AHB挂载告诉外设，总线的最低地址成为总线基地址.

  1. 总线基地址

 总线基地址即为外设Block2的基地址，为ox4000 0000.

总线名称总线基地址相对外设基地址偏移
APB10x4000 00000x0
APB20x4001 00000x0001 0000
AHB0x4001 80000x0001 8000
  1. 外设基地址

 参考STM32F1xx中文参考手册第28页，以下为GPIO的外设基地址;

外设名称外设基地址相对APB2总线基地址偏移
GPIOA0x4001 08000x0000 0800
GPIOB0x4001 0C000x0000 0C00
GPIOC0x4001 10000x0000 1000
GPIOD0x4001 14000x0000 1400
GPIOE0x4001 18000x0000 1800
GPIOF0x4001 1C000x0000 1C00
GPIOG0x4001 20000x0000 2000

 APB2总线基地址为0x4001 0000.

  1. 外设寄存器地址

 每个寄存器为32bit，占用4字节，按照顺序排列在外设及地址上，寄存器地址用相对于外设基地址的偏移地址来描述，以下为GPIOB寄存器为例(GPIOB基地址为0x4001 0C00)，可参考STM32F1xx中文参考手册的8.5章节:

寄存器名称寄存器地址相对外设基地址偏移
GPIOB_CRL0x4001 0C000x00
GPIOB_CRH0x4001 0C040x04
GPIOB_IDR0x4001 0C080x08
GPIOB_ODR0x4001 0C0C0x0C
GPIOB_BSRR0x4001 0C100x10
GPIOB_BRR0x4001 0C140x14
GPIOB_LCKR0x4001 0C180x18

C语言封装寄存器

 比如，控制GPIOB端口的第5个管脚输出低电平。首先需要指导GPIOB是在哪个总线，然后根据总线基地址及偏移地址得到GPIO外设基地址，最后通过外设基地址和寄存器偏移得到寄存器基地址;

  1. 总线和外设基地址封装
// 定义外设基地址，对应Block2即Peripherals的基地址
#define PREIPH_BASE         ((unsigned int)0x40000000)
// 定义APB2总线基地址，由Block2基地址 + APB2偏移即0x10000
#define APB2PERIPH_BASE     (PREIPH_BASE + 0x00010000)
// 定义GPIOB外设基地址
#define GPIOB_BASE          (APB2PERIPH_BASE + 0x0C00)
// 定义寄存器基地址
#define GPIOB_CRL           *(unsigned int*)(GPIOB_BASE+0x00)
#define GPIOB_CRH           *(unsigned int*)(GPIOB_BASE+0x04)
#define GPIOB_IDR           *(unsigned int*)(GPIOB_BASE+0x08)
#define GPIOB_ODR           *(unsigned int*)(GPIOB_BASE+0x0C)
#define GPIOB_BSRR          *(unsigned int*)(GPIOB_BASE+0x10)
#define GPIOB_BRR           *(unsigned int*)(GPIOB_BASE+0x14)
#define GPIOB_LCKR          *(unsigned int*)(GPIOB_BASE+0x18)

 如果向GPIOB第5个管脚输出低电平.首先找到端口输出数据寄存器，功能说明可通过GPIOx_BSRR来对各ODR位进行设置;

 然后找到端口设置/清除寄存器，参考下图功能说明，若需GPIOB第5管脚输出低电平(图中清除对应的ODRy为0)，需设置高16位的BRy为1(这里y位5，即将0x01<<(16+5)赋给GPIOB_BSRR指向的内容)，注意不要设置BSy，否则只有BSy起作用;若需GPIOB第5管脚输出高电平(图中设置对应的ODRy位为1)，需设置低16位的BSy为1(即0x01<<5赋给GPIOB_BSRR指向的内容);

 参考代码如下:

*(unsigned int*)GPIOB_BSRR = (0x01<<(16+5));    // GPIO第5个管脚输出低电平
*(unsigned int*)GPIOB_BSRR = (0x01<<5);         // GPIO第5个管脚输出高电平

 由于GPIOx_ODR可读可写，因此可直接设置GPIOx_ODR对应位的值，

#define setBit(v, n) ((v) |= ((unsigned int)1 << (n)))  //将v的第n位置1
#define clrBit(v, n) ((v) &= (~(unsigned int)1 << (n))) //将v的第n位清0
clrbit(*(unsigned int*)GPIOB_ODR, 5) // GPIO第5个管脚输出低电平
setbit(*(unsigned int*)GPIOB_ODR, 5) // GPIO第5个管脚输出高电平

 同理，如果需要读GPIOB第5管脚的电平高低，需参考端口输入数据寄存器，如下图:

 首先需读取GPIOx_IDR寄存器的内容，然后取其第5位的值，即该管脚的状态，代码如下:

#define getBit(v, n) ((v) & ((unsigned int)1 << (n)))   //获取v的第n位
unsigned int tmp1 = *(unsigned int*)GPIOB_IDR;   // 读取GPIOB寄存器I/O状态
int tmp2 = getBit(tmp1, 5) >> 5     // 读取GPIOB第5管脚状态
  1. 寄存器封装

 由于寄存器具有相似的功能，地址具有相似性，为了更方便的访问寄存器，由于结构体成员是连续存放，可通过结构体进行封装，代码如下:

typedef unsigned int uint32_t;
typedef unsigned short uint16_t;
typedef struct {
    uint32_t CRL;   // 0x00
    uint32_t CRH;   // 0x04
    uint32_t IDR;   // 0x08
    uint32_t ODR;   // 0x0C
    uint32_t BSRR;  // 0x10
    uint32_t BRR;   // 0x14
    uint16_t LCKR;  // 0x18
} GPIO_TYPE;

 此时如果需要完成GPIOB第5管脚输出低电平，代码如下:

GPIO_TYPE *GPIOx;
GPIOx = (GPIO_TYPE*)GPIOB_BASE
GPIOx->BSRR = 1<<(16+5)

 以上是根据存储器映射找到外设地址、再找寄存器地址，并介绍了外设是如何封装的，后续程序编写将直接采用ST公司提供的库.
