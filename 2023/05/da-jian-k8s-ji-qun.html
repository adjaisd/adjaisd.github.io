搭建k8s集群

 k8s版本采用1.27.2版本，网络插件采用calico，集群具有1个master节点、2个node节点;

 最开始采用containerd，即便配置了config.toml但拉取镜像仍太慢，最后采用cri-dockerd作为k8s的CRI.

环境准备

  1. 配置ssh免密登录，修改/etc/hosts与hostname

 配置免密，分别登录每个节点，修改/etc/ssh/sshd_config与~/.ssh/authorized_keys，通过ssh-keyscan将公钥添加known_hosts内容，过程略；

 master节点执行以下指令:

# 编辑/etc/hosts
cat > /etc/hosts << EOF
127.0.0.1 localhost
192.168.10.171 master1
192.168.10.172 node01
192.168.10.173 node02
EOF
# 修改hostname
name="master1" && hostnamectl set-hostname ${name}
nodes="node01 node02"
for name in ${nodes}; do
  scp /etc/hosts ${name}:/etc/hosts
  ssh ${name} "hostnamectl set-hostname ${name}"
done
  1. 安装docker，cri-dockerd，并修改配置

 所有节点都执行;

# 安装docker并修改配置
apt update && apt install curl -y
curl -fsSL https://get.docker.com | bash -s docker --mirror Aliyun
mkdir -p /etc/docker && cat > /etc/docker/daemon.json << EOF
{
  "log-driver": "json-file",
  "log-opts": {
    "max-size": "100m",
    "max-file": "3"
  },
  "exec-opts": ["native.cgroupdriver=systemd"],
  "max-concurrent-downloads": 10,
  "max-concurrent-uploads": 10,
  "registry-mirrors": [
    "https://registry.cn-hangzhou.aliyuncs.com",
    "https://mirror.ccs.tencentyun.com",
    "https://registry.docker-cn.com",
    "https://hub-mirror.c.163.com",
    "https://dockerhub.azk8s.cn",
    "https://gcr.azk8s.cn",
    "https://quay.azk8s.cn"
  ]
}
EOF
systemctl daemon-reload && systemctl restart docker

 master节点执行;

# 安装cri-dockerd，先下载(可用迅雷) https://github.com/Mirantis/cri-dockerd/releases
wget https://github.com/Mirantis/cri-dockerd/releases/download/v0.3.2/cri-dockerd-0.3.2.amd64.tgz -P /tmp
ls /tmp | grep cri-dockerd
tar -zxf /tmp/cri-dockerd-0.3.2.amd64.tgz -C /tmp/ && ls /tmp
cp /tmp/cri-dockerd/cri-dockerd /usr/bin/
chmod +x /usr/bin/cri-dockerd

# 配置cri-dockerd自启动
cat > /usr/lib/systemd/system/cri-docker.service << EOF
[Unit]
Description=CRI Interface for Docker Application Container Engine
Documentation=https://docs.mirantis.com
After=network-online.target firewalld.service docker.service
Wants=network-online.target
Requires=cri-docker.socket

[Service]
Type=notify
ExecStart=/usr/bin/cri-dockerd --container-runtime-endpoint fd:// --network-plugin=cni --pod-infra-container-image=registry.cn-hangzhou.aliyuncs.com/google_containers/pause:3.9
ExecReload=/bin/kill -s HUP \$MAINPID
TimeoutSec=0
RestartSec=2
Restart=always
StartLimitBurst=3
StartLimitInterval=60s
LimitNOFILE=infinity
LimitNPROC=infinity
LimitCORE=infinity
TasksMax=infinity
Delegate=yes
KillMode=process
[Install]
WantedBy=multi-user.target
EOF

# 生成socket 文件
cat > /usr/lib/systemd/system/cri-docker.socket << EOF
[Unit]
Description=CRI Docker Socket for the API
PartOf=cri-docker.service
[Socket]
ListenStream=%t/cri-dockerd.sock
SocketMode=0660
SocketUser=root
SocketGroup=docker
[Install]
WantedBy=sockets.target
EOF
# 启动cri-docker并设置开机自动启动
systemctl daemon-reload && systemctl enable cri-docker --now
systemctl is-active cri-docker
# 修改containerd配置，用于k8s自带的crictl指令
cat > /etc/crictl.yaml << EOF
runtime-endpoint: unix:///run/cri-dockerd.sock
image-endpoint: unix:///run/cri-dockerd.sock
timeout: 10
debug: false
EOF
systemctl restart containerd

 以上安装完成docker与cri-dockerd后，在master节点执行:

# 同步文件与配置到节点
nodes="node01 node02"
files="
/usr/bin/cri-dockerd
/usr/lib/systemd/system/cri-docker.service
/usr/lib/systemd/system/cri-docker.socket
/etc/crictl.yaml
"
for name in ${nodes}; do
    for file in ${files}; do
      scp ${file} ${name}:${file}
    done
    ssh ${name} "systemctl daemon-reload && systemctl enable cri-docker --now && systemctl is-active cri-docker"
    ssh ${name} "systemctl restart containerd"
done
  1. k8s准备环境，包括关闭swap分区，启用桥接，转发

 所有节点执行

swapoff -a # 临时禁用swap分区
# 配置k8s允许桥接、转发
cat > /etc/modules-load.d/k8s.conf << EOF
overlay
br_netfilter
EOF
modprobe overlay && modprobe br_netfilter
# 配置允许桥接转发
cat > /etc/sysctl.d/k8s.conf << EOF
net.bridge.bridge-nf-call-ip6tables = 1
net.bridge.bridge-nf-call-iptables  = 1
net.ipv4.ip_forward                 = 1
EOF
sysctl --system
lsmod | grep -E "(br_netfilter)|(overlay)"
  1. 添加k8s源，安装kubeadm，kubelet，kubectl

 所有节点执行;

# 添加k8s仓库（ubuntu22）
apt-get update && apt-get install -y apt-transport-https
curl https://mirrors.aliyun.com/kubernetes/apt/doc/apt-key.gpg | apt-key add -
cat > /etc/apt/sources.list.d/kubernetes.list << EOF
deb https://mirrors.aliyun.com/kubernetes/apt/ kubernetes-xenial main
EOF
# 安装kubelet, kubeadm, kubectl
apt-get update
apt-get install -y kubelet kubeadm kubectl
apt-mark hold kubelet kubeadm kubectl
  1. kubeadm初始化

 master节点执行;

kubeadm init \
  --image-repository=registry.cn-hangzhou.aliyuncs.com/google_containers \
  --service-cidr=10.96.0.0/16 \
  --pod-network-cidr=192.168.0.0/16 \
  --cri-socket=/run/cri-dockerd.sock \
  --control-plane-endpoint=master1 \
  --apiserver-advertise-address=192.168.10.171

mkdir -p ~/.kube
cp -i /etc/kubernetes/admin.conf ~/.kube/config
chown "$(id -u):$(id -g)" ~/.kube/config

kubectl get nodes
  1. 安装网络插件calico

 master节点执行;

wget https://raw.githubusercontent.com/projectcalico/calico/v3.25.1/manifests/tigera-operator.yaml
wget https://raw.githubusercontent.com/projectcalico/calico/v3.25.1/manifests/custom-resources.yaml
kubectl create -f tigera-operator.yaml
kubectl create -f custom-resources.yaml
watch kubectl get pods -n calico-system # 验证

  1. 节点加入网络

 master执行;

# kubeadm token create --print-join-command # 获取节点需执行加入网络的指令
tmp="$(kubeadm token create --print-join-command) --cri-socket=/run/cri-dockerd.sock"
nodes="node01 node02"
for name in ${nodes}; do
  ssh ${name} ${tmp}
done

  1. 镜像加速

 k8s在1.24版本后，不再采用dockershm作为容器运行时环境，而是采用containerd，如果想支持docker，可考虑cri-dockerd;

 containerd配置较麻烦，命令也较麻烦，配置文件需要修改/etc/containerd/config.toml;

 此外，在通过containerd下载calico3.25.1所需镜像，速度非常慢，采用docker就快了很多;

 下面是节点成功加入且状态都已Ready时，master与node节点所需的镜像(由于docker操作不了部分镜像，可以采用crictl指令);

 至此，k8s1.27.2集群部署完毕.

  1. 采用本地registry加速

 有时镜像下载太慢，可以先在vps上下载好镜像并上传到registry，采用服务商的容器镜像服务，由于命名空间是唯一的，需解决镜像tag的命名问题;

 或者采用官方的registry镜像，作为本地镜像registry，需修改/etc/docker/daemon.json，添加"insecure-registries"字段，参考如下:

mkdir -p /etc/docker && cat > /etc/docker/daemon.json << EOF
{
  "exec-opts": ["native.cgroupdriver=systemd"],
  "max-concurrent-downloads": 10,
  "max-concurrent-uploads": 10,
  "registry-mirrors": [
    "https://registry.cn-hangzhou.aliyuncs.com",
    "https://mirror.ccs.tencentyun.com",
    "https://registry.docker-cn.com",
    "https://hub-mirror.c.163.com",
    "https://dockerhub.azk8s.cn",
    "https://gcr.azk8s.cn",
    "https://quay.azk8s.cn"
  ],
  "insecure-registries": [
    "192.168.10.171:5000"
  ]
}
EOF
systemctl daemon-reload && systemctl restart docker

 然后需要启动registry镜像，docker run -d -p 5000:5000 -v /opt/data/registry:/var/lib/registry docker.io/registry，并开放端口;

 采用本地registry最好在内网，如果速度实在慢，也可通过公网;

 搭建本地registry教程很多，但批量tag与push镜像的几乎没有，下面为批量tag和push到本地registry的脚本;

 手动tag、push镜像实在费力，下面是批量tag与push镜像的脚本，可自行修改:

images="$(crictl images | awk '{printf("%s:%s\n",$1, $2)}')"
prefix="192.168.10.171:5000"
for image in ${images}; do
  arr=(${image//:/ }) # 按照':'切割，右边的为得到版本号
  ver=${arr[1]}       # 版本
  name=${arr[0]}      # image名字
  arr1=(${name//\// })  # 按照'/'切割，得到url、namespace和name
  len=${#arr1[@]}     # 按照'/'切割后的长度，url可为空，长度为2或3
  # 镜像包含前缀，跳过
  [[ ${image} =~ ^"${prefix}".* ]] && continue
  if [ ${len} == 2 ]; then    # 无前缀url
    new_tag="${prefix}/${arr1[0]}/${arr1[1]}:${ver}"
  elif [ ${len} == 3 ]; then  # 有前缀url
    new_tag="${prefix}/${arr1[1]}/${arr1[2]}:${ver}"
  else continue
  fi
  docker tag "${image}" "${new_tag}"
  docker push "${new_tag}"
  echo "${new_tag}"
done
  1. 安装calicoctl工具

 官网有很多方法，包括作为单独容器、作为pod、作为kubectl的插件等;

 这里采用其二进制文件，将其放入/usr/local/bin目录下;

wget -L https://github.com/projectcalico/calico/releases/latest/download/calicoctl-linux-amd64 -P /tmp
cp /tmp/calicoctl-linux-amd64 /usr/local/bin/calicoctl
chmod +x /usr/local/bin/calicoctl
# 拷贝到节点
nodes="node01 node02" && for name in ${nodes}; do scp /usr/local/bin/calicoctl ${name}:/usr/local/bin/calicoctl; done

  1. 新增控制平面

 新的控制平面为master2节点，需在master1节点执行以下指令：

# master节点作为控制平面加入网络
tmp="$(kubeadm token create --print-join-command) \
  --certificate-key $(kubeadm init phase upload-certs --upload-certs | tail -n 1) \
  --cri-socket=/run/cri-dockerd.sock \
  --control-plane"
nodes="master2" && for name in ${nodes}; do
  ssh "${name}" "${tmp}"
  ssh "${name}" "rm ~/.kube -rf && mkdir -p ~/.kube"
  scp ~/.kube/config ${name}:~/.kube/config
done
