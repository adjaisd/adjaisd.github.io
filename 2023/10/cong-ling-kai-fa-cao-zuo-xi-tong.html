从零开发操作系统

 近两个月在日常固定的学习工作之余，抽空弥补了下大学期间缺失的课程学习，偏计算机底层的编程语言-汇编语言，顺便了解下操作系统的启动;

 之后学习操作系统可能会持续很长的一段时间，可能要好几年，也可能坚持不了那么久便中断了，为此应该记录下期间的学习过程.

原因

 事实上，从大二学计算机四大件时，其中数据结构和计算机网络还好，尽管还是缺少了很多实践并且漏掉了些内容，但起码可以通过上手实践弄懂，而至于计算机组成原理和操作系统，由于学校条件限制，计组因为偏硬件，只能通过特定的仿真软件(软件叫EngineDream)进行实验，当时时间太紧迫了，照着实验报告步骤一步一步做，可能当时懂，但现在全忘了;

 操作系统同样如此，印象深刻的是，当时实验环境是在服务器上，服务器因为高并发导致无法实验，实验课都无了，但由于老师和我关系较近，后面我做了部分实验并录制了视频(本人的shell编程能力得到很好的锻炼)，尽管期末考试通过了，但仍然存在很多疑问，由于缺少了实验，并且课程几乎算是纯理论了，很多知识到如今也差不多忘了;

 对比B站公开课，确实差了不少，缺少实操，我相信很多人课上完了，相关知识会随着时间过去而遗忘，即便再次回顾，尽管有熟悉感，但也不清楚其中具体细节;

 由于本人最初(高一)接触的便是汇编语言，并且买了王爽的《汇编语言》，所谓”书非借不能读也”，当初遗漏的也应该拾起来了，另外如果之后做底层开发或相关研究，有必要了解其中的具体细节.

汇编语言学习

准备工作

  1. 16位程序运行环境doxbox，网上很多安装教程略，或者采用开源项目doxbox-x;

    • 关于dosbox-x如何自动挂载?

     在dosbox-x的安装目录下打开doxbox-x.conf，找到[autoexec]，示例如下:

    [autoexec]
mount X: xxx\MASM           # 将宿主机的'xxx\MASM'挂载到dosbox的'X:'
mount Y: xxx\masm -t dir    # 将宿主机的'xxx\masm'类型挂载到'Y:'
                            # 其中'-t dir'为挂载类型，若存在目录，也可以挂载到dosbox中
Y:                          # 切换到'Y:'路径下
    • 如何为’masm.exe’添加环境变量?

     同样，打开doxbox-x.conf，找到[config]，在它的后面几行找到set path选项，改为如下配置:

    # 在自动挂载配置中，已经将masm的安装目录挂载到了'X:'，需要将'X:'添加到环境变量，用封号分开
set path    = X:\;Z:\;Z:\SYSTEM;Z:\BIN;Z:\DOS;Z:\4DOS;Z:\DEBUG;Z:\TEXTUTIL

  2. 集成开发环境比如VScode、或者Clion专业版，安装略.

参考教程

 汇编语言只需达到入门程度就够了，之后gdb时遇到不会的再查资料;

 推荐参考B站视频《汇编语言》速成指南(全程敲代码);

 切记全程务必敲代码，在后面练习中建议多写几遍，思路不难，中间可能有些细节需要注意，编译不通过也不要直接看代码，多尝试几次，错误都遇到了才清楚有哪些要注意.

操作系统-BootLoader阶段一

 这里采用x86架构，非其他架构，当然在其他架构下也可以进行开发，这里通过开源的模拟器qemu进行学习;

 Boot是机器启动时第一个执行的程序，负责硬件自检、加载BIOS、加载系统内核、加载驱动等，当前Boot的目标是在屏幕输出一段字符串;

准备工作

  1. 安装VMware，过程略

  2. 创建虚拟机，系统为ubuntu22.04，且虚拟机安装gcc、gdb、nasm、make、cmake等，过程略

  3. 虚拟机安装qemu，参考官方仓库，或者通过apt-get install qemu-system安装所有架构下的模拟器;

  4. 宿主机安装Clion专业版，过程略

Boot实现

 教程可以参考os-tutorial-cn;

 此外，也可参考《自己动手写操作系统》、《从实模式到保护模式》.
