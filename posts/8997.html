Linux入门

Linux程序

  • Linux应用程序表现为两种特殊类型的文件：可执行文件和脚本文件。
    • 可执行文件是计算机可以直接运行的程序，相当于Windows中的.exe文件；
    • 脚本文件是一组指令的集合，这些指令将由另一个程序(即解释器)来执行，相当于Windows中的.bat文件、.cmd文件或解释执行的BASIC程序。
  • 搜索路径
    • 可以由系统管理员配置，通常包含以下存储系统程序的标准路径。
      • /bin:二进制文件目录，用于存放启动系统时用到的程序；
      • /usr/bin:用户二进制文件目录，存放用户使用的标准程序；
      • /usr/local/bin:本地二进制文件，存放软件安装的程序。
    • 系统管理员(如root用户)登录后使用的PATH变量可能还包含存放系统系统管理程序的目录，如 /sbin和 /usr/sbin。
    • 可选的操作系统和第三方应用程序可能被安装在/opt目录下，安装程序可以通过用户安装脚本将路径添加到PATH环境变量中。
    • 环境变量PATH：Linux与UNIX一样，采用冒号(: )分隔PATH变量里的条目，而不像MS-DOS和Windows使用分号(; )，下面给出一个例子
      /usr/local/bin:/bin:/usr/bin:.:/home/neil/bin:/usr/X11R6/bin
       这里PATH中包含了标准程序存放目录、当前目录(. )、home目录和X视窗系统目录。

注意:
 Linux采用正斜线(/)分隔文件名里的目录名，而不是像Windows那样用反斜线(/)。而且这次还是UNIX的用法在先

C语言编译器

  • 书中使用的是gcc编译器，就按照书上的来吧…

  • 实验：第一个Linux C语言程序

    • 第一步，编写代码

      #include<stdio.h>
#include<stdlib.h>

int main(){
  printf("Hello World\n");
  exit(0);
}

    • 第二步，编译、链接和运行程序

gcc -o hello hello.c
./hello

注意

  • 关于 exit(0) 与 return 0 在C/C++中的区别可以参考blog；
  • 执行编译后的程序，比如这里为编译为hello，执行时会优先搜索PATH里的每一个路径，假如恰巧存在一个hello文件，那么当前目录下的hello就无法执行成功，建议通过 ./hello 执行；
  • 关于-o name参数，如果略去，gcc会把程序放在一个名为a.out文件里(assembler output，即汇编输出)。

头文件

  • 在Windows中，我们常常发现include目录下有许多c语言编写的头文件，而在Linux系统下，一般头文件可以在/usr/include目录及其子目录中，或者/usr/include/sys和/usr/include/linux中找到
  • 如果想包含一个自定义的头文件，可以采用-I path参数，示例如下：
gcc -I/usr/openwin/include fred.c

 这里通过-I参数告诉gcc可以从标准位置，也可以从/usr/openwin/include目录中查找gred.c中包含的头文件。

库文件

 库文件是一组预先编译好的函数的集合，这些函数都是按照可重用的原则编写的。它们通常由一组相互关联的函数组成以执行某项常见的任务，比如屏幕处理函数库(curses和ncurses库)；
 库文件的名字总是以lib开头，随后的部分指明这是什么库(例如，c代表C语言库，m代表数学库)，文件名的最后部分以.开始，然后给出库文件的类型：

  • .a代表传统的静态函数库；
  • .so代表共享函数库
gcc -o fred fred.c /usr/lib/libm.a    // 在 /usr/lib/lbm.a中查找所需的库
gcc -o fred fred.c -lm                // 在 标准库目录中查找名为libm库，-l指定了具体的名字，比如-lx11，那么就找libx11
gcc -o fred fred.c -L/usr/openwin/lib //除了在标准库目录，gcc会在/usr/openwin/lib中查找需要的库

注意

  • 如果找到了两个同名但后缀不同的库，会优先选择共享库，即优先选择.so后缀的库

静态库

 静态库，也称为归档文件(archive)，按照惯例它们的文件名都以.a结尾；当程序需要使用函数库中某个函数时，它包含一个声明该函数的头文件；还需要使用-l选项指明除标准c语言运行库外还要使用的库。
 可以通过ar(代表archive，即建立归档文件)和gcc -c命令对函数分别进行编译，下面给出一段示例：

 在本例中，将创建一个小型函数库，它包含两个函数，然后将在一个示例程序中调用其中一个函数，这两个函数分别是fred和bill，它们只打印欢迎信息。

  1. 首先，建立两个源文件，分别命名为fred.c和bill.c，源文件内容分别如下

第一个源文件fred.c

#include<stdio.h>
void fred(int arg){
  printf("fred: we passed %d\n", arg);
}

第二个源文件bill.c

#include<stdio.h>
void bill(char *arg){
  printf("bill: we passed %s\n", arg);
}
  1. 可以分别编译这些函数以产生要包含在库文件中的目标文件。由于没有main函数，无法创建完整的程序。
gcc -c bill.c fred.c      // 会生成bill.o与fred.o文件
  1. 接着编写一个调用bill函数的程序，需要建立一个库，这里建的库名为lib.h，包含了对函数的声明
void bill(char *);
void fred(int);
  1. 定义主函数，文件名为program.c
#include<stdlib.h>
#include "lib.h"
int main(){
  bill("Hello world");
  exit(0);
}
  1. 然后编译主函数文件，链接先前编译好的目标模块bill.o
gcc -c program.c          // 会生成program.o文件
gcc -o program program.o bill.o   // 将program.o与用到的bill.o文件链接
./program                 // 执行program
  1. 通过建立一个库文件，取名为libfoo.a
ar crv libfoo.a bill.o fred.o

在某些操作系统下，要想成功使用这个函数库，需要通过ranlib命令为函数库生成一个内容表；如果使用Linux的GNU的软件开发工具，这一步不是必须的，做了也无妨。

  1. 通过使用函数库，来创建program程序
gcc -o program program.o libfoo.a   // 调用库函数创建程序
./program               // 执行program

 可以使用-L选择具体目录下，-l选择具体的库，比如

gcc -o program program.o -L. -lfoo

 -L.告诉gcc从当前目录(.)寻找函数库，-lfoo告诉gcc要找名为libfoo.a的函数库。

共享库

 静态库有个缺点，当同时运行许多程序并且都用来自同一个函数库的函数时，内存中就会有同一函数的多份副本，而且在程序文件自身中也有多份同样的副本。
 共享库在不同操作系统上有不同的实现，书上介绍不多，具体可以查找网上资源。

 可以通过ldd命令查看程序所要用到的共享库。

ldd program

 Linux下的.so文件类似Windows下的.dll文件。
